# XRPL EVM Binary Update Guide

This guide will help you update your XRPL EVM node to the latest version.

## Prerequisites

- Existing XRPL EVM node installation
- Root or sudo access
- Basic knowledge of Linux commands

## Update Steps

### 1. Check Current Version

```bash
xrpld version
```

### 2. Stop the Service

```bash
sudo systemctl stop xrpld
```

### 3. Backup Important Files

⚠️ **Important**: Always backup your validator key and node configuration before updating!

```bash
# Backup validator key
cp $HOME/.xrpl/config/priv_validator_key.json $HOME/priv_validator_key.json.backup

# Backup node key
cp $HOME/.xrpl/config/node_key.json $HOME/node_key.json.backup
```

### 4. Download and Install New Binary

#### Option A: Download Pre-built Binary (Recommended)

```bash
cd $HOME
wget -O xrpld.tar.gz "$(curl -s https://api.github.com/repos/xrplevm/node/releases/latest | jq -r '.assets[] | select(.name | contains("linux") and (contains("amd64") or contains("x86_64"))) | .browser_download_url' | head -1)"
tar -xzf xrpld.tar.gz
rm -f install.sh xrpld.tar.gz
chmod +x xrpld
sudo mv xrpld /usr/local/bin/
[ -f "libwasmvm.x86_64.so" ] && sudo mv libwasmvm.x86_64.so /usr/lib/
[ -f "libwasmvm.so" ] && sudo mv libwasmvm.so /usr/lib/
```

#### Option B: Build from Source

```bash
# Clone repository
cd $HOME
git clone https://github.com/xrplevm/node
cd $(basename https://github.com/xrplevm/node .git)

# Checkout recommended version
git checkout v8.0.0

# Build binary
make install

# Verify installation
$(basename https://github.com/xrplevm/node .git)d version
```

### 5. Verify New Version

```bash
xrpld version
```

The output should show the new version number.

### 6. Start the Service

```bash
sudo systemctl start xrpld
```

### 7. Check Logs

```bash
sudo journalctl -u xrpld -f -o cat
```

### 8. Verify Node is Running

```bash
xrpld status 2>&1 | jq .SyncInfo
```

## Rollback (If Needed)

If you encounter issues with the new version, you can rollback to the previous version:

### 1. Stop the Service

```bash
sudo systemctl stop xrpld
```

### 2. Restore Previous Binary

If you kept a backup of the old binary:

```bash
sudo cp /usr/local/bin/xrpld.backup /usr/local/bin/xrpld
```

Or reinstall the previous version:

```bash
cd $HOME/node
git checkout v1.0.0
make install
```

### 3. Start the Service

```bash
sudo systemctl start xrpld
```

## Upgrade with Cosmovisor (Automated)

If you're using Cosmovisor for automated upgrades:

### 1. Create Upgrade Directory

```bash
mkdir -p $HOME/.xrpl/cosmovisor/upgrades/upgrade-8.0.0/bin
```

### 2. Download New Binary

```bash
cd $HOME
wget -O xrpld.tar.gz "$(curl -s https://api.github.com/repos/xrplevm/node/releases/latest | jq -r '.assets[] | select(.name | contains("linux") and (contains("amd64") or contains("x86_64"))) | .browser_download_url' | head -1)"
tar -xzf xrpld.tar.gz
chmod +x xrpld
mv xrpld $HOME/.xrpl/cosmovisor/upgrades/upgrade-8.0.0/bin/
```

### 3. Verify Binary

```bash
$HOME/.xrpl/cosmovisor/upgrades/upgrade-8.0.0/bin/xrpld version
```

Cosmovisor will automatically switch to the new binary at the specified upgrade height.

## Common Issues

### Issue: Binary not found after update

**Solution**: Make sure the binary is in your PATH:

```bash
which xrpld
```

If not found, check if it's in `/usr/local/bin/` or `$HOME/go/bin/`

### Issue: Version mismatch

**Solution**: Clear the old binary and reinstall:

```bash
sudo rm $(which xrpld)
# Then reinstall using one of the methods above
```

### Issue: Node won't start after update

**Solution**: Check the logs for errors:

```bash
sudo journalctl -u xrpld -n 100 --no-pager
```

Common fixes:
- Reset the node data (if not a validator): `xrpld tendermint unsafe-reset-all --home $HOME/.xrpl`
- Check if the genesis file is correct
- Verify peer connections

### Issue: Sync issues after update

**Solution**: Try state sync or snapshot:

```bash
# Stop service
sudo systemctl stop xrpld

# Reset data
xrpld tendermint unsafe-reset-all --home $HOME/.xrpl --keep-addr-book

# Use state sync (see State Sync guide)
# Or download snapshot (see Snapshot guide)

# Start service
sudo systemctl start xrpld
```
